## 录屏时操作顺序（3 分钟）

### 1. 开场（10 秒）
- 桌面 → 打开项目文件夹 → 展示文件结构

### 2. 实时信号（30 秒）
- 终端 → `python3 main.py` → 等待输出完成

### 3. 日报展示（20 秒）
- Finder → `reports/` → 打开日报 → 滚动浏览

### 4. 风控代码（30 秒）
- VS Code → `risk_control.py` → 滚动到关键函数 → 停留 5 秒

### 5. 回测演示（30 秒）
- 终端 → `python3 backtest_demo.py` → 展示结果

### 6. 配置文件（20 秒）
- VS Code → `config.yaml` → 展示配置项

### 7. Docker（20 秒）
- 终端 → `cat docker-compose.yml` → 展示配置

### 8. GitHub（20 秒）
- 浏览器 → GitHub 仓库 → 滚动 README → 展示 commits

### 9. 结束（10 秒）
- 回到桌面 → 显示项目名称

## 录屏后处理

### 剪辑工具推荐
- **macOS**: iMovie（免费）
- **跨平台**: 剪映（免费，易用）
- **专业**: Final Cut Pro / Adobe Premiere

### 必做处理
1. 删除等待时间（如程序加载）
2. 添加字幕（关键功能说明）
3. 检查视频大小（< 30MB）

### 导出设置
- 格式: MP4 (H.264)
- 分辨率: 1920×1080 或 1280×720
- 帧率: 30fps
- 比特率: 5-8 Mbps

## 常见问题

### Q: 录屏时程序报错怎么办？
A: 提前多测试几次，确保环境稳定。如果还是报错，可以用**预录制的截图**代替。

### Q: 视频太大超过 30MB 怎么办？
A: 降低分辨率到 1280×720，或者压缩视频（用 HandBrake）。

### Q: 没有 Telegram Bot 怎么办？
A: 跳过 Telegram 推送演示，或者用**模拟截图**展示效果。

### Q: 回测太慢怎么办？
A: 用 `backtest_demo.py` 脚本（已生成），输出模拟结果，只需 2 秒。
